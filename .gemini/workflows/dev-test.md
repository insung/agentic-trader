# Agentic Trader: Development & Test Workflow (/dev-test)

이 워크플로우는 코드를 수정하거나 새로운 기능을 추가한 후, 전체 파이프라인이 정상적으로 동작하는지 검증하기 위한 CI/CD 자동화 지침서입니다.
AI 에이전트는 아래 절차를 순차적으로 수행합니다.

// turbo-all

## 1. 패키지 종속성 검증
가상환경 내 필요한 모든 패키지가 설치되어 있는지 확인합니다.
Run: `make install`

## 2. 단위 및 통합 테스트 실행
LLM 및 파이프라인의 에러가 없는지 `pytest`를 실행하여 확인합니다.
Run: `make test`

## 3. 결과 분석 및 피드백
- **모두 통과 (Passed):** "테스트가 100% 통과했습니다. 커밋(Commit)할까요?"라고 사용자에게 물어봅니다.
- **실패 (Failed):** 테스트 로그를 분석하여 원인을 진단하고, 어떤 코드를 어떻게 고쳐야 할지 사용자에게 **수정안(diff)**을 제시합니다. 사용자가 승인하면 즉시 코드를 고치고 다시 2번 단계로 돌아가 테스트를 반복합니다.
